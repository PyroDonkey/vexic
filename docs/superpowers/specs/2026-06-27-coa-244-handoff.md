## What to build (from the spec)

1. `console/lib/control-plane-client.mjs` (new): thin `fetch` wrapper to the
   hosted control-plane. One function per console op. Owns base URL, the
   `Authorization: Bearer ${VEXIC_CONTROL_PLANE_TOKEN}` header, usage
   normalization, and HTTP-status -> error mapping.
2. `console/lib/control-plane-store.mjs` (rewrite as dispatcher): same exported
   function names as today. Select real client vs the preserved in-memory stub
   per the dispatch rules below. Keep `resetStoreForTests`.
3. `console/lib/control-plane-api.mjs` (minimal): add `await` to the store calls.
   No signature or error-shape change. Leave `requireUser`/`requireOrg` guards
   exactly as they are.
4. `console/tests/control-plane-client.test.mjs` (new): mock `fetch`, assert
   method/path/headers/body per op, the DELETE->POST-revoke mapping, usage
   normalization (incl. cost formatting), and the status->error table.
5. `console/.env.example`: add `VEXIC_CONTROL_PLANE_URL` and
   `VEXIC_CONTROL_PLANE_TOKEN`.

### Endpoint mapping (`/control/v1/clerk-orgs/{orgId}/...`)

| Console fn | HTTP | Notes |
| --- | --- | --- |
| listProjects | GET /projects | |
| createProject | POST /projects | body {name, environment} |
| getProject | GET /projects/{pid} | |
| listAgentKeys | GET /projects/{pid}/keys | |
| createAgentKey | POST /projects/{pid}/keys | returns {rawKey, key} |
| revokeAgentKey | POST /projects/{pid}/keys/{kid}/revoke | console DELETE -> server POST-revoke; 204 |
| usageSummary | GET /projects/{pid}/usage | normalize totals (see below) |
| supportMetadata | (no server endpoint) | return empty when wired |

`orgId` comes only from server-side `readAuthContext()` (Clerk `session.orgId`).
Never from anything browser-controllable.

## Decisions already made (do not re-litigate)

- Dispatch/fallback: URL set -> real client (missing token -> request still sent,
  server 401, fail closed; never silently fall back to stub). URL unset AND
  `NODE_ENV !== "production"` -> stub. URL unset AND `NODE_ENV === "production"`
  -> fail closed with a clear "not configured" error.
- Error mapping (client layer only; never the console-layer guards): 400->400
  `invalid_request`; 404->404 `not_found`; upstream 401/403->500
  `control_plane_unavailable`; network/timeout->502. Log upstream status
  server-side for 401/403/5xx.
- Usage normalization: server returns totals keyed `requests, writes, retrievals,
  modelRequests, inputTokens, outputTokens, totalTokens, estimatedCostMicros`,
  with `caps: {}`. Display `requests`, `writes`, `retrievals`, plus
  `estimatedCostMicros` formatted as dollars (micros / 1e6, labeled as cost).
  Omit `modelRequests` and the token metrics for MVP. Leave `caps` as-is (UI
  renders "No cap").
- Server credential token env var (already implemented): `VEXIC_CONTROL_PLANE_TOKENS`
  (comma-separated) must include the console token.
